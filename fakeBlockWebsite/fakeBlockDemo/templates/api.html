FakeBlock Public API Documentation

POST /api/fake/

Decides if data provided in request body parameters contain fake news or not. If both body parameters `post_text` and `news_text` are provided, the result will reflect the truth value of evaluating truth(post_text) or truth(news_text). Neither parameter is required, and either can be provided.

This API expects and responds with a content type of application/json.

Body Parameters
post_text:	(OPTIONAL) string, contains the plain text body of the Facebook post to evaluate 
news_text:	(OPTIONAL) string, contains the plain text of the news title banner attached to the Facebook post to evaluate
JSON Response Fields
fake:		boolean, whether overall post was determined to contain fake news.
confidence:	float, the certainty of the post being fake. 
		(Range 0-1, 0.0 being certainty of post being real, 1.0 being certainty of post being fake.)
Example Usage

The following is a simple cURL example that makes a POST request to this API with all possible body parameters.

curl -XPOST -H "Content-type: application/json" -d '{"post_text":"string1", "news_text":"string2"}' 'https://fakeblocker.herokuapp.com/api/fake'

OPTIONS /api/fake/

This method is for handling CORS preflight requests. This is a public API, so all origins are allowed. It will always return a JSON response containing the below headers only: 


headers = {
	'Access-Control-Allow-Origin': "*",
	'Access-Control-Allow-Methods': ["POST", "OPTIONS"],
	'Access-Control-Allow-Headers': [
		'accept',
		'accept-encoding',
		'authorization',
		'content-type',
		'dnt',
		'origin',
		'user-agent',
		'x-csrftoken',
		'x-requested-with',
	],
	'Content-Type': "application/json",
}
